# 交易纪律与复盘方法论

StockTracker 的交易记录是已成交事实账本，不承担交易前计划或下单决策功能。

交易纪律能力沉淀在 AI 分析和对话里：当用户询问“交易没章法”“某笔买卖是否合理”“道氏理论是什么”或要求复盘时，Agent 会用统一方法论组织回答，而不是要求用户在每笔成交记录里维护计划字段。

## 复盘维度

| 维度 | 关注点 |
| --- | --- |
| 事实账本 | 成交日期、买卖方向、价格、数量、费用、分红和备注。 |
| 成本收益 | FIFO 单笔已实现盈亏、券商摊薄口径持仓成本、累计收益和当前行情盈亏。 |
| 仓位风险 | 交易后仓位变化、是否降低集中度、是否扩大回撤暴露。 |
| 行情位置 | 当前价格相对支撑、阻力、均线、估值和近期波动的位置。 |
| 行为纪律 | 是否追涨杀跌、是否频繁补仓、是否把累计结果误读成单笔决策质量。 |

## 常见交易框架

| 框架 | 用法 |
| --- | --- |
| 道氏理论 | 分析趋势结构、阶段高低点、量价确认和趋势反转信号；它更偏技术分析框架，不是单一技术指标。 |
| 趋势跟随 | 判断是否顺着已形成的趋势交易，以及趋势破坏后是否需要退出或降仓。 |
| 均值回归 | 分析超跌、低估或偏离均值后的修复交易，重点检查是否有回归依据和失效条件。 |
| 基本面投资 | 分析估值、业绩、现金流、行业格局和长期竞争力是否支撑交易。 |
| 股息现金流 | 分析分红、派息、现金流回报和长期持有成本是否匹配。 |
| 资产配置 | 分析组合权重、再平衡、市场分散和风险预算。 |
| 风险控制 | 分析仓位、最大亏损、止损条件、回撤和单一标的暴露。 |

## Agent 回答口径

Agent 复盘交易时遵循以下边界：

1. 交易记录只代表已发生事实，不推断用户当时的计划。
2. 回答必须区分已发生交易事实、单笔结果、累计收益和当前行情。
3. 当前行情或技术指标只能用于事后复盘，不能倒推出交易发生当天的依据。
4. 用户明确提到某个理论或方法时，Agent 再按该框架展开；否则默认使用通用复盘维度。
