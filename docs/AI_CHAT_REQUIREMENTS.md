# AI 对话功能需求文档

## 1. 背景与目标

StockTracker 当前已支持 AI 组合分析、个股分析和大盘分析。新功能将在现有 AI 配置和本地 SQLite 持久化能力之上，增加一个可随时唤起的 AI 对话入口，让用户围绕当前持仓和相关股票数据进行连续问答。

该功能定位为“股票投资相关的个人理财/投研对话助手”，不是通用聊天机器人。AI 必须以个人理财专家身份工作，并且只回答与用户当前持仓、用户明确提到的股票、交易记录、成本、盈亏、风险、估值、行情和相关市场信息有关的问题。

## 2. 用户价值

- 用户无需离开当前页面，即可询问自己持仓相关问题。
- 用户可以基于自己的真实持仓、交易记录和个股基础数据获得更贴近上下文的解释。
- 用户可以在轻量浮窗中快速提问，也可以进入 AI 页面进行完整对话。
- 对话内容持久化保存，便于后续回看、继续上下文或清理。

## 3. 功能范围

### 3.1 AI 对话定位

系统提示词固定包含以下核心角色与边界：

- AI 是个人理财专家，面向个人投资者提供股票投资相关分析。
- AI 的回答必须基于用户当前系统中的持仓、交易记录、股票基础数据、自动抓取到的未持仓股票数据和可获得的行情/估值信息。
- AI 只能回答与股票资产、持仓、交易、风险、估值、行情和资产配置相关的问题。
- 对无关问题必须拒答，并引导用户回到持仓、股票、交易、风险或资产配置相关主题。
- AI 不能给出收益承诺、内幕信息、确定性买卖指令或规避风险提示的表达。
- AI 输出应保留“仅供参考，不构成投资建议”的边界。

### 3.2 数据权限

AI 对话可读取以下数据：

- 当前持仓列表：股票代码、名称、市场、备注。
- 每只股票的交易记录：买入、卖出、分红、价格、数量、费用、日期、备注。
- 计算型持仓摘要：当前持仓数量、平均成本、已实现盈亏、未实现盈亏、总盈亏、手续费、分红等。
- 个股基础数据：当前价格、涨跌幅、PE、PB、EPS、市值等现有行情接口可返回的数据。
- 技术指标数据：如现有个股详情页使用的 K 线和技术指标快照，首版可按需接入。
- 未持仓股票数据：当用户询问未在当前持仓中的股票时，系统应根据股票代码/市场自动抓取可用的行情、估值和基础数据。
- AI 配置：provider、baseUrl、model、temperature、maxContextTokens、语言等。

AI 对话默认不读取以下数据：

- 与当前投资资产无关的浏览器数据或本机文件。
- API Key 明文。
- 与股票无关的私人信息。

### 3.3 对话范围控制

用户问题分为三类：

- 允许回答：与当前持仓、某只已记录股票、交易复盘、成本、风险、估值、行情、组合结构、仓位管理相关。
- 条件回答：与未持仓股票相关的问题允许回答。系统需要自动抓取该股票可用的行情、估值和基础数据；AI 需说明该标的不在当前持仓中，并区分“系统持仓事实”和“外部抓取数据”。
- 拒绝回答：天气、编程、娱乐、医疗、法律、通用百科、与股票资产无关的闲聊等。

拒答时应简短说明范围限制，并提供可继续提问的方向，例如：

> 我只能回答与你当前持仓和股票投资相关的问题。你可以问我“当前组合最大风险是什么”或“帮我复盘某只股票的交易节奏”。

### 3.4 上下文管理

系统需要支持上下文长度统计和清理：

- 上下文长度在 UI 中展示为用户友好的等级，例如“短 / 中 / 长 / 接近上限”，不直接暴露精确 token 数作为主要信息。
- 可在 tooltip 或辅助信息中显示估算 token 数，便于排查问题。
- 显示接近上限的提醒，例如“上下文较长，建议清理或新建会话”。
- 支持清空当前会话消息。
- 支持新建会话。
- 支持删除历史会话。
- 首版 token 统计可采用近似估算：中文按字符数、英文按单词和符号估算；后续可替换为 tokenizer。
- 发送请求时应限制传入模型的历史消息数量，避免超过 `aiConfig.maxContextTokens` 和模型上下文窗口。

### 3.5 AI Token 配置

AI 对话只对用户暴露上下文长度配置，不再提供单次输出 token 配置，避免用户把输出长度和上下文窗口混淆。

- 字段名：`maxContextTokens`
- 中文名称：最大上下文长度
- 默认值：`128000`，即 128K token
- 最小值：`4096`
- 用途：限制每次发送给模型的 system prompt、持仓摘要、外部抓取数据、历史消息和当前用户问题的总上下文长度。

设置页仅展示：

- `Max Context Tokens` / `最大上下文长度`：控制一次对话请求最多携带多少上下文。

当估算上下文超过 `maxContextTokens` 时，系统应自动裁剪较早历史消息，并保留：

- system prompt
- 当前用户问题
- 当前持仓/相关股票必要摘要
- 最近若干轮对话
- 未持仓股票自动抓取到的必要数据

### 3.6 持久化

所有对话内容需保存到本地 SQLite，按当前本机 `userId` 隔离。

建议新增两类数据：

- `ai_chat_sessions`
  - `id`
  - `user_id`
  - `title`
  - `scope`
  - `created_at`
  - `updated_at`

- `ai_chat_messages`
  - `id`
  - `session_id`
  - `user_id`
  - `role`: `system | user | assistant`
  - `content`
  - `context_snapshot_json`
  - `token_estimate`
  - `created_at`

说明：

- `system` 消息可以不直接展示给用户，但建议保存或可重建。
- `context_snapshot_json` 用于记录当次回答所引用的持仓/股票摘要，避免未来持仓变化后历史回答失去依据。
- 清空当前会话时，可删除消息或标记会话已清空；首版建议直接删除该会话下消息。
- 对话历史暂不纳入 JSON 导出/导入备份。
- 用户执行“清空数据”时，需要同步清空 AI 对话会话与消息历史。

## 4. 交互设计

### 4.1 全局悬浮入口

在主应用布局中增加一个全局悬浮小球：

- 常驻固定在页面右下角。
- 在桌面端保持不遮挡主要按钮和底部内容。
- 在移动端适当缩小，并保留安全边距。
- 点击后打开轻量 AI 对话浮窗。
- 当 AI 未启用或配置不完整时，点击后显示配置引导，而不是直接报错。
- 页面滚动时，小球自动收起，减少对内容阅读和操作的遮挡。
- 页面停止滚动后，小球再次浮现。
- 滚动停止判断可使用短延迟 debounce，首版建议 300-500ms。

悬浮入口状态：

- 默认：显示 AI 图标。
- 收起：滚动过程中缩小、降低可见面积或贴边隐藏，仅保留弱提示，不阻塞页面交互。
- 浮现：滚动停止后平滑回到默认状态。
- 加载中：发送消息时显示轻微 loading 状态。
- 错误：最近一次请求失败时可用弱提示，不做强干扰。

### 4.2 轻量对话浮窗

点击悬浮小球后打开对话浮窗：

- 浮窗用于快速提问和查看最近上下文。
- 默认尺寸适合桌面右下角使用。
- 顶部包含标题、上下文长度、清理按钮、放大按钮、关闭按钮。
- 中间为消息列表。
- 底部为输入框和发送按钮。
- 支持 `Enter` 发送，`Shift + Enter` 换行。
- 发送中禁用重复提交，并展示 assistant 正在生成的状态。

浮窗内需要提供：

- 当前会话标题或“新对话”。
- 上下文长度统计。
- 清理当前会话按钮。
- 放大按钮。
- AI 配置缺失提示。
- 空状态建议问题。

### 4.3 放大与跳转

点击浮窗放大按钮后：

- 自动跳转至新增独立路由 `/ai/chat`。
- 当前会话 ID 需要随路由或状态传递，保证进入全屏后继续同一段对话。
- 如果没有当前会话，则进入 AI 对话区并创建/展示默认新会话。

建议路径：

- 首版新增独立 `/ai/chat` 路由，作为完整对话页面。
- 现有 `/ai` 页面保留“分析中心”定位，并增加进入 `/ai/chat` 的入口。

### 4.4 `/ai/chat` 全屏对话体验

在新增 `/ai/chat` 页面中提供完整对话区：

- 左侧或顶部显示会话列表。
- 主区域显示完整消息流。
- 支持新建会话、切换会话、删除会话、清空当前会话。
- 支持查看上下文长度。
- 上下文长度展示为“短 / 中 / 长 / 接近上限”等友好等级；必要时可补充估算 token。
- 支持较长输入和多轮回答。
- 支持从浮窗无缝延续。
- 支持流式输出，assistant 回复应逐步展示，而不是等待完整结果一次性出现。

全屏页面应优先服务深度对话：

- 消息列表可滚动。
- 输入框固定在底部。
- 长回答保持可读宽度。
- 历史会话标题自动根据第一条用户消息生成，首版可取前 20 个字符。

## 5. 关键状态

### 5.1 默认状态

- AI 已启用且配置完整。
- 展示最近一次会话或新对话。
- 输入框可用。

### 5.2 空状态

无历史消息时展示建议问题：

- “当前组合最大的风险是什么？”
- “帮我复盘最近交易最多的股票。”
- “我当前持仓里哪只股票需要重点关注？”
- “按成本和盈亏帮我总结一下仓位结构。”

### 5.3 配置缺失

当 AI 未启用或模型配置不完整：

- 浮窗和全屏对话均显示配置提示。
- 提供进入设置页的入口。
- 禁止发送消息。

### 5.4 无持仓

当当前没有持仓：

- AI 仍可说明“当前没有持仓数据”。
- 引导用户先添加股票或询问如何规划记录结构。
- 不进行具体个股建议。

### 5.5 发送中

- 输入框保持可见。
- 发送按钮 disabled。
- 消息区出现 assistant loading 气泡。
- 请求失败后保留用户消息，并展示可重试入口。

### 5.6 超出上下文

- 显示上下文过长提示。
- 建议清理当前会话或新建会话。
- 请求时自动截断较早消息，但不得截断当前用户问题和必要持仓摘要。

### 5.7 拒答

- 对无关问题给出简短拒答。
- 不调用外部无关工具。
- 不进入长篇解释。

## 6. API 与服务设计

建议新增接口：

- `GET /api/ai/chat/sessions`
  - 获取当前用户会话列表。

- `POST /api/ai/chat/sessions`
  - 新建会话。

- `DELETE /api/ai/chat/sessions`
  - 删除会话。

- `GET /api/ai/chat/messages?sessionId=...`
  - 获取某个会话的消息。

- `POST /api/ai/chat`
  - 发送用户消息，保存用户消息，构建上下文，流式调用模型，边生成边返回 assistant 回复，并在完成后保存 assistant 回复。

- `DELETE /api/ai/chat/messages`
  - 清空某个会话消息。

服务层建议新增：

- `lib/ai/chat.ts`
  - 构建 system prompt。
  - 判断问题是否在股票范围内。
  - 汇总持仓上下文。
  - 为未持仓股票自动抓取可用行情、估值和基础数据。
  - 构建模型 messages。
  - 以 streaming 方式调用 provider。
  - 估算 token。

- `lib/sqlite/db.ts`
  - 增加 chat session/message 的 CRUD。

## 7. System Prompt 草案

```text
你是 StockTracker 内置的个人理财专家，服务对象是正在管理自己股票、基金、港股、美股、A 股或加密资产记录的个人投资者。

你只能回答与用户当前持仓、用户明确提到的股票、交易记录、股票基础数据、估值、行情、风险、仓位、复盘和资产配置有关的问题。

如果用户询问与股票投资无关的内容，你必须礼貌拒绝，并引导用户回到持仓、交易复盘、股票估值、行情或风险管理相关问题。

你可以基于系统提供的持仓数据、交易记录、盈亏摘要、个股基础数据、技术指标，以及系统为未持仓股票自动抓取到的可用数据进行分析。你不能编造系统未提供的数据；如果数据不足，必须明确说明。

你不能承诺收益，不能声称确定涨跌，不能提供内幕消息，不能把回答包装成绝对买卖指令。你的回答应以风险、条件、证据和可验证信号为中心。

默认使用中文回答。回答需要具体、直接、可执行，但必须保留“仅供参考，不构成投资建议”的边界。
```

## 8. 自动补全的产品口径

以下是根据现有系统和本次 review 已确认的产品口径：

- 模型连接配置优先使用服务端 `.env.local`：`AI_PROVIDER`、`AI_BASE_URL`、`AI_MODEL`、`AI_API_KEY`。
- 如果 `.env.local` 未配置完整，则继续使用设置页中的本地兜底配置。
- 设置页继续控制 Temperature、Max Context Tokens、新闻增强和分析语言；提示词由 Skill/Task 固定维护，不再从设置页编辑。
- README 与 `.env.example` 必须说明每个环境变量的用途、可选值、示例值和是否会暴露到前端。
- 使用当前本机 `userId` 进行会话隔离，保持本地优先，不引入登录体系。
- 首版必须支持流式输出，这是基础体验要求。
- 首版不做真正 tokenizer，先实现估算 token 统计。
- UI 中上下文长度展示为“短 / 中 / 长 / 接近上限”等友好等级。
- 首版不支持附件上传。
- 首版不支持语音输入。
- 首版不支持跨设备同步。
- 首版允许回答未持仓股票问题；如回答需要数据，系统需要自动抓取可用数据，并明确标注“未在当前持仓中”。
- 未持仓股票自动抓取时，如果用户没有提供市场且无法唯一匹配，弹出候选项让用户选择具体标的。
- AI 设置需要新增“最大上下文长度”配置项，与现有“最大输出长度”分开。
- 对话历史暂不纳入 JSON 导出/导入备份。
- 清空数据时必须同时清空 AI 对话历史。
- 全屏对话区使用新增独立路由 `/ai/chat`。
- 清空会话需要二次确认，使用现有统一 `ConfirmDialog`。
- 删除会话也需要二次确认。

## 9. 验收标准

- 所有页面右下角出现 AI 悬浮入口。
- 页面滚动时悬浮入口会自动收起，停止滚动后再次浮现。
- 点击悬浮入口可以打开轻量对话浮窗。
- 在 AI 配置完整时可以发送与持仓相关的问题并获得回答。
- AI 可以读取当前持仓和个股基础数据生成上下文。
- 对未持仓股票问题，系统可以自动抓取可用行情/估值/基础数据并回答。
- AI 对无关问题会拒答。
- AI 回复支持流式输出。
- 对话消息刷新页面后仍存在。
- 可以查看上下文长度友好等级，例如“短 / 中 / 长 / 接近上限”。
- 可以在设置页配置最大上下文长度。
- 可以清空当前会话内容。
- 可以从浮窗点击放大进入 `/ai/chat` 完整对话区。
- 全屏对话区可以继续同一会话。
- JSON 导出/导入暂不包含 AI 对话历史。
- 清空数据会同时清空 AI 对话历史。
- `pnpm test` 和 `pnpm build` 通过。

## 10. 已确认的取舍

- 首版暂不需要会话列表搜索。
- 首版不需要用户手动选择本次对话使用哪些持仓或股票，系统自动判断和组装上下文。
- 上下文长度展示为“短 / 中 / 长 / 接近上限”等友好等级。
- 未持仓股票自动抓取时，如果用户没有提供市场且无法唯一匹配，弹出候选项让用户选择具体标的。
